# 07. 铁铺 (TiePM) 使用指南

> **铁铺**是玄铁语言的官方包管理器。提供搜索、安装、发布铁器（玄铁包）的完整工作流。

---

## 7.1 概述

### 7.1.1 什么是铁铺？

铁铺（TiePM）之于玄铁，犹如 npm 之于 Node.js、cargo 之于 Rust。它解决了以下问题：

- **发现**：搜索可用的铁器（库/工具）
- **安装**：一键安装铁器到本地，自动验证完整性
- **发布**：将你的项目打包并登记到官方索引，供他人使用

### 7.1.2 核心概念

| 术语 | 说明 |
|------|------|
| **铁器** | 可发布的玄铁包（库或可执行程序） |
| **索引** | 远程 JSON 文件，记录所有可用铁器的元信息 |
| **tiepm.toml** | 项目根目录下的包清单文件，描述铁器的元信息 |
| **哈希** | SHA256 校验值，确保下载文件完整且未被篡改 |

### 7.1.3 工作目录

铁铺的所有本地数据存储在用户目录下：

```
~/.tiepm/
├── 缓存/              # 下载的 .zip 缓存
├── 已安装/            # 已解压安装的铁器
│   ├── 数组工具/
│   ├── 网络库/
│   └── ...
└── 索引.json          # 本地缓存的远程索引
```

## 7.2 安装铁铺

铁铺本身也是一个铁器，可以通过自举方式安装。

### 方式一：从源码编译

```bash
# 克隆铁铺仓库
git clone https://github.com/markjy/铁铺.git
cd 铁铺

# 使用玄铁编译器编译
玄铁 铁 铁铺.xt
```

编译产物为 `铁铺.exe`，将其放入 PATH 目录即可全局使用。

### 方式二：使用玄铁编译器直接执行（0.18.x暂不支持）

```bash
玄铁 跑 铁铺.xt -- 安装 <包名>
```

## 7.3 命令参考

### 7.3.1 安装 — `铁铺 安装 <包名>`

从远程索引查找并安装指定铁器。

```bash
铁铺 安装 数组工具
```

**执行流程**：

1. 从 `https://tiepm.xt.markjy.com/index.json` 获取远程索引
2. 查找包名，获取最新版本的哈希、仓库地址
3. 构造下载 URL：`{仓库}/archive/v{版本}.zip`
4. 下载 ZIP 到 `~/.tiepm/缓存/`
5. 使用 `certutil -hashfile` 计算 SHA256 并与索引记录比对
6. 解压到 `~/.tiepm/已安装/<包名>/`
7. 清理下载缓存

**别名**：`铁铺 install <包名>`（英文命令等效）

### 7.3.2 搜索 — `铁铺 搜索 <关键词>`

在远程索引中按关键词搜索铁器，匹配包名和描述。

```bash
铁铺 搜索 网络
```

输出示例：

```
=== 搜索结果 ===
网络库 v1.2.0 - HTTP 客户端与服务端工具
数组工具 v1.0.0 - 提供数组去重、过滤、映射等工具函数
```

**别名**：`铁铺 search <关键词>`

### 7.3.3 发布 — `铁铺 发布`

将当前目录的项目打包并输出发布所需信息。

**前置条件**：项目根目录存在 `tiepm.toml` 文件。

```bash
铁铺 发布
```

**执行流程**：

1. 读取并解析 `tiepm.toml`
2. 使用 `tar -czf` 打包项目（排除 `.git`、`.tiepm`、打包文件自身）
3. 计算 `.zip` 文件的 SHA256
4. 输出发布信息和 API 调用指令

**输出示例**：

```
[铁铺] === 发布铁器 ===
[铁铺] 包名: 数组工具 v1.0.0
[铁铺] SHA256: a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6...

=== 发布信息 ===
包名: 数组工具
版本: 1.0.0
哈希: a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6...
仓库: https://github.com/username/array-utils

发布步骤:
1. git tag v1.0.0 && git push --tags
2. 创建 GitHub Release 并上传 数组工具_v1.0.0.zip
3. curl -X POST https://tiepm.xt.markjy.com/api/publish \
   -H "Content-Type: application/json" \
   -d '{"名称":"数组工具","版本":"1.0.0","哈希":"...",\
        "仓库":"https://github.com/username/array-utils"}'
```

**完整发布工作流**：

```bash
# 1. 确保代码已提交
git add . && git commit -m "v1.0.0 发布准备"

# 2. 执行铁铺发布（打包 + 计算哈希）
铁铺 发布

# 3. 打 Git 标签并推送
git tag v1.0.0
git push origin master --tags

# 4. 在 GitHub 上创建 Release，上传输出的 .zip 文件

# 5. 调用 API 登记到索引
curl -X POST https://tiepm.xt.markjy.com/api/publish \
  -H "Content-Type: application/json" \
  -d '{"名称":"数组工具","版本":"1.0.0","哈希":"...","仓库":"..."}'
```

**别名**：`铁铺 publish`

### 7.3.4 列表 — `铁铺 列表`

列出所有已安装的铁器。

```bash
铁铺 列表
```

输出示例：

```
[铁铺] === 已安装的铁器 ===
  数组工具
  网络库
```

**别名**：`铁铺 list`

### 7.3.5 信息 — `铁铺 信息 <包名>`

查看指定铁器的详细信息（版本、仓库、哈希）。

```bash
铁铺 信息 数组工具
```

输出示例：

```
=== 数组工具 v1.0.0 ===
仓库: https://github.com/username/array-utils
SHA256: a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6...
```

**别名**：`铁铺 info <包名>`

### 7.3.6 清理 — `铁铺 清理`

清理 `~/.tiepm/缓存/` 目录中的下载缓存。

```bash
铁铺 清理
```

**别名**：`铁铺 clean`

### 7.3.7 帮助 — `铁铺 帮助`

显示帮助信息。

```bash
铁铺 帮助
# 或
铁铺 -h
铁铺 --help
```

## 7.4 包清单：tiepm.toml

每个铁器项目根目录需要一个 `tiepm.toml` 文件来描述包的元信息。

### 7.4.1 完整模板

```toml
# tiepm.toml - 铁铺包清单

[包]
名称 = "数组工具"                              # 必填：包名
版本 = "1.0.0"                                 # 必填：语义版本号
仓库 = "https://github.com/username/array-utils" # 必填：GitHub 仓库地址

[作者]
姓名 = "开发者名"                              # 选填
邮箱 = "dev@example.com"                       # 选填

[依赖]
# 铁铺 = "2.0.0"                               # 预留：依赖声明（后续支持）

[描述]
简介 = "提供数组去重、过滤、映射等工具函数"     # 选填：功能简述
许可 = "MIT"                                   # 选填：许可证
```

### 7.4.2 字段说明

**[包]** 节（必填）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 名称 | 字 | 是 | 铁器名称，用于搜索和安装 |
| 版本 | 字 | 是 | 语义版本号，格式 `主.次.补` |
| 仓库 | 字 | 是 | GitHub 仓库完整 URL |

**[作者]** 节（选填）：

| 字段 | 类型 | 说明 |
|------|------|------|
| 姓名 | 字 | 作者/维护者名称 |
| 邮箱 | 字 | 联系邮箱 |

**[依赖]** 节（预留）：

当前版本不处理依赖关系。未来版本将支持声明对其他铁器的版本依赖。

**[描述]** 节（选填）：

| 字段 | 类型 | 说明 |
|------|------|------|
| 简介 | 字 | 包功能一句话描述，用于搜索结果展示 |
| 许可 | 字 | 开源许可证名称 |

## 7.5 版本规范

铁铺采用语义版本号（Semantic Versioning）：

```
主版本.次版本.补丁版本
  MAJOR . MINOR . PATCH
```

| 版本变更 | 含义 |
|----------|------|
| 补丁版本 +1 | 向后兼容的 bug 修复（`1.0.0` → `1.0.1`） |
| 次版本 +1 | 向后兼容的功能新增（`1.0.0` → `1.1.0`） |
| 主版本 +1 | 不兼容的 API 变更（`1.0.0` → `2.0.0`） |

安装时默认获取最新版本（按字符串排序比较版本号）。

## 7.6 索引工作原理

### 7.6.1 索引格式

远程索引是一个 JSON 对象，位于 `https://tiepm.xt.markjy.com/index.json`：

```json
{
  "数组工具": {
    "1.0.0": {
      "版本": "1.0.0",
      "哈希": "a1b2c3d4e5f6...",
      "仓库": "https://github.com/username/array-utils",
      "作者": "张三",
      "描述": "提供数组去重、过滤、映射等工具函数"
    }
  }
}
```

每个包名下可以包含多个版本。搜索 API 返回每个包的最新版本。

### 7.6.2 下载地址构造

铁铺从 GitHub Releases 下载包文件。下载 URL 由仓库地址和版本号拼接而成：

```
{仓库}/archive/v{版本}.zip
```

例如：
- 仓库：`https://github.com/username/array-utils`
- 版本：`1.0.0`
- 下载 URL：`https://github.com/username/array-utils/archive/v1.0.0.zip`

这个 URL 由 GitHub 自动生成（基于 Git tag `v1.0.0`），无需手动上传 Release 资源。

### 7.6.3 本地缓存

首次安装时，铁铺从远程获取索引并缓存到 `~/.tiepm/索引.json`。后续操作优先使用缓存，减少网络请求。

## 7.7 安全机制

### 7.7.1 SHA256 哈希验证

每个包在索引中记录了其 `.zip` 文件的 SHA256 哈希值。安装时铁铺会：

1. 下载 `.zip` 文件到本地缓存
2. 计算实际文件的 SHA256
3. 与索引中记录的哈希比对
4. 不匹配则拒绝安装并报错

这确保了文件在传输过程中未被篡改或损坏。

### 7.7.2 手动验证（可选）

你也可以手动验证下载的文件：

```bash
# Windows (certutil)
certutil -hashfile 数组工具_v1.0.0.zip SHA256

# Linux/macOS
sha256sum 数组工具_v1.0.0.zip
```

## 7.8 创建你的第一个铁器

本节通过一个完整示例演示从零开始创建并发布一个铁器。

### 第一步：创建项目

```bash
mkdir 我的工具
cd 我的工具
```

### 第二步：编写源码

创建 `工具.xt`：

```
// 工具.xt - 我的第一个铁器

公 函 问好(名称) {
    返 "你好，" & 名称 & "！"
}

公 函 加法(a, b) {
    返 a + b
}
```

### 第三步：创建清单

创建 `tiepm.toml`：

```toml
[包]
名称 = "我的工具"
版本 = "1.0.0"
仓库 = "https://github.com/你的用户名/我的工具"

[描述]
简介 = "一个简单的示例铁器"
```

### 第四步：发布

```bash
# 执行发布命令
铁铺 发布

# 按照输出的步骤：
# 1. 初始化 Git 仓库并推送
git init
git add .
git commit -m "v1.0.0"
git tag v1.0.0
git remote add origin https://github.com/你的用户名/我的工具.git
git push -u origin master --tags

# 2. 调用 API 登记
curl -X POST https://tiepm.xt.markjy.com/api/publish \
  -H "Content-Type: application/json" \
  -d '{"名称":"我的工具","版本":"1.0.0","哈希":"<从发布命令输出复制>","仓库":"https://github.com/你的用户名/我的工具","描述":"一个简单的示例铁器"}'
```

### 第五步：验证

```bash
# 换一个目录，尝试安装刚发布的铁器
cd ..
铁铺 安装 我的工具
```
