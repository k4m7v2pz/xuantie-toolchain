## v0.17.6 (2026-05-04)
修复编译赋值标量存储bug，16_控制流测试全通过

- 修复了 `编译赋值` 对标量变量存储原始值而非标记值（tagged pointer）的严重bug：当执行 `x = 1` 时，编译器生成 `store i64 1` 而非 `store i64 3`（tagged(1)），导致后续读取和比较全部失效。此bug的传导链为：赋值存raw值 → 读取时被当作tagged值右移1位 → 数值错误 → 条件判断失败 → `续`语句永远不触发。修复方案：将标量存储分支的 `此.符(翻译右[0])` 改为 `xt_val`，与 `编译变量声明语句` 保持一致；
- 16_控制流测试全部5项通过：续跳过奇数、断跳出循环、嵌套断隔离、嵌套续隔离、当+遍历嵌套断；
- 回归测试 05/06/07 全部通过，fib 性能无退化；

## v0.17.5 (2026-05-04)——fib(30) 6ms 超越 Go
自举编译器类型注解驱动优化,fib(30) 6ms 超越 Go

- 新增了自举编译器的 `映射类型注解` 函数：将 `整`→`raw_i64`、`小数`→`double`、`逻`→`i1`，为参数脱壳和静态优化提供统一映射；
- 新增了自举编译器编译函数定义的参数脱壳：读取解析器存入的 `arg节点.推导类型`（如 `"整"`），对标量参数先 `ashr` 脱壳再存储为 raw_i64，跳过不必要的 `xt_retain` 和 `追踪对象`；
- 新增了 `函数返回类型表`：在编译函数定义时注册返回值类型（如 `函 xxx(): 整`），调用处自动 `ashr` 脱壳为 raw_i64 供后续原生运算使用；
- 新增了编译中缀的标量快速路径：当 `+`/`-`/`*`/`/` 等运算符的左右操作数都为标量类型（`raw_i64`/`i1`/`double`）时，跳过动态分发（`xt_dict_get` + `xt_string_new`），直接走原生 LLVM 指令；
- 修复了编译赋值对标量变量存储标记值的问题：`k = k + 1` 现在对 raw_i64 变量存储原始值而非标记值，与种子编译器修复同源；
- 新增了解析器 `是保留关键字` 检测函数：识别 `结果`、`任务`、`道`、`整`、`字`、`逻` 等 15 个关键字，在变量声明、遍历循环、函数定义等位置拦截撞名用法并输出中文友好报错（与种子编译器行为对齐），修复了 07/08/09/15 等测试的解析失败问题；
- 修复了自引用报错：`预扫描引入` 检测到文件引用自身时调用 `终 1` 输出中文错误并立即终止，取代之前的静默跳过；
- 修复了 `出作用域` 无限循环崩溃的严重 bug：根因是种子编译器的 `compileForStatement` 缺少 `continueLabels` 和 `breakLabels` 的 push/pop，导致 `续`/`断` 在嵌套循环中跳转到错误标签。修复方案：在条件分支后提前生成 `stepLabel`，同步 push `breakLabels(endLabel)` + `continueLabels(stepLabel)` + `loopDepths` 三个栈，循环结束后 pop。修复后 05/06 编译正常，新增 16_控制流测试专项覆盖；
- **[性能突破] 斐波那契基准测试**：带注解 `斐波那契(n: 整): 整` 的 fib(30) 从 365ms 降至 **~6ms**（约 60 倍提升），超越 Go 的 ~15ms 和种子编译器的 ~18ms，标志着自举代码生成质量达到生产级；

## v0.17.3 (2026-05-03)

- 新增了种子编译器的函数类型注解支持：编译器现在能识别 `函 xxx(n: 整): 整` 中的参数和返回类型注解，将整数参数以 raw_i64 形式存储，跳过不必要的装箱和 ARC 操作；

- 优化了字符串核心操作的编译方式，把 `字节数`、`取字节`、`字符` 这些常用方法直接硬编码进后端，避开了自举时容易踩坑的动态查找；
- 优化了符号生成安全性，增加了 `符()` 和 `标()` 包装函数，彻底封锁了“空”值被误当成字符串塞进 IR 的漏洞；
- 优化了跨模块的类型解析，现在支持继承别的模块里的类（`型 A 承 别名.B`），实例化也支持带前缀的写法了；
